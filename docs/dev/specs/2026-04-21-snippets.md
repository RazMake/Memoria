# PRD: Snippets — Text Expansion for Markdown

**Date**: 2026-04-21
**Status**: Draft

## 1. TL;DR

Add a new Memoria feature — **Snippets** — that lets users define reusable text expansions triggered via VS Code's autocomplete in Markdown files and the Todo Editor popup. Snippets can be **static** (constant text) or **dynamic** (TypeScript functions receiving parameters and file context). Blueprints define the snippet folder path and ship their own set of snippet files; users can add or override snippets in the same folder. A special `@` prefix triggers **contact snippets** that pull data from the Contacts feature. Snippet functions that throw produce inline error messages. Snippets marked as path-safe remain active even when the snippets feature is disabled.

---

## 2. Goals / Non-goals

**Goals**
- Provide a `CompletionItemProvider`-based text expansion system for Markdown files within initialized Memoria workspaces.
- Support static snippets (constant string output) and dynamic snippets (TypeScript functions with parameters).
- Support contact-based snippets via the `@` prefix, pulling data from `ContactsFeature`.
- Provide a `HoverProvider` for inserted contact snippet text that displays all contact properties on hover.
- Allow parameter selection via QuickPick cascade for multi-parameter snippets.
- Allow single-parameter snippets to accept inline parameters from selected text: user selects text, types `{`, and the selection becomes the parameter value.
- Load user-authored `.ts` snippet files at runtime by compiling and importing them dynamically.
- Each snippet declares a glob pattern controlling which files it appears in — the `CompletionItemProvider` filters suggestions by glob.
- Snippet functions receive the current file's `TextDocument` and cursor `Position`, enabling file-context-aware expansions.
- Snippets work identically in normal Markdown editing and in the Todo Editor add/edit popup (webview autocomplete).
- Snippets marked `pathSafe: true` remain functional for programmatic path/filename expansion even when the snippets feature is disabled.
- Blueprint-shipped snippets can be reset to their original version via a context menu entry.
- When a snippet function throws, the error message is inserted inline **and** a VS Code warning notification is shown.

**Non-goals**
- Snippet syncing across machines or team sharing.
- Multi-cursor snippet expansion.
- Snippet marketplace or community repository.
- Snippet chaining (one snippet invoking another).
- Rich snippet editing UI (snippets are authored as `.ts` files in a code editor).
- Supporting non-Markdown file types.
- Replacing VS Code's built-in snippet system — Memoria snippets are domain-specific text expansions, not tab-stop/placeholder snippets.

---

## 3. User-visible behavior

### 3.1 Feature registration

- Feature ID: `snippets`. Appears in `Memoria: Manage Features`; enable/disable is authoritative via `.memoria/features.json`.
- Blueprint feature entry type: `SnippetsFeatureEntry` — declares `snippetsFolder` (e.g. `"_Snippets/"`).
- The snippet folder path is stored in the blueprint manifest (`.memoria/blueprint.json`) under `snippets.snippetsFolder` and is not user-configurable post-init.
- No new activation events needed — the extension already activates via `workspaceContains:.memoria/blueprint.json`.

### 3.2 Snippet folder structure

The snippet folder (e.g. `_Snippets/`) lives at the workspace root. It contains `.ts` files, each exporting one or more snippet definitions. Blueprints ship initial snippet files into this folder at workspace initialization; users can add their own `.ts` files to the same folder.

```
_Snippets/
├── date-time.ts        ← blueprint-shipped
├── team.ts             ← blueprint-shipped
├── my-custom.ts        ← user-created
```

**Blueprint-shipped vs. user-created**: The file manifest in `.memoria/blueprint.json` tracks which files were shipped by the blueprint (via content hashes). Files present in the manifest are "blueprint-owned"; files not in the manifest are user-created. This reuses the existing `fileManifest` mechanism from `BlueprintManifest`.

### 3.3 Snippet file format

Every `.ts` file in the snippet folder must `export default` an array of `SnippetDefinition` objects (or a single object). The extension compiles and loads these at feature start and on file change.

```typescript
import type { SnippetDefinition, SnippetContext } from "memoria-snippets";

const snippets: SnippetDefinition[] = [
    // Static snippet — no function, just a label and text
    {
        trigger: "{pp}",
        label: "Private Preview",
        description: "Inserts 'private preview'",
        glob: "**/*.md",
        body: "private preview",
    },
    {
        trigger: "{PP}",
        label: "Public Preview",
        description: "Inserts 'Public Preview'",
        glob: "**/*.md",
        body: "Public Preview",
    },
    // Dynamic snippet with parameters
    {
        trigger: "{date}",
        label: "Date",
        description: "Inserts the current date in the chosen format",
        glob: "**/*.md",
        parameters: [
            { name: "format", options: ["YYYY-MM-dd", "MM/dd/YYYY", "dd MMM YYYY", "YYYY"] },
        ],
        expand(ctx: SnippetContext): string {
            const fmt = ctx.params.format ?? "YYYY-MM-dd";
            const now = new Date();
            // ... format logic ...
            return formatted;
        },
    },
    // Dynamic snippet using file context
    {
        trigger: "{heading}",
        label: "Current heading",
        description: "Inserts the nearest heading above the cursor",
        glob: "**/*.md",
        expand(ctx: SnippetContext): string {
            const lines = ctx.document.getText().split("\n");
            for (let i = ctx.position.line; i >= 0; i--) {
                if (lines[i].startsWith("# ")) return lines[i].slice(2);
            }
            return "(no heading found)";
        },
    },
];

export default snippets;
```

### 3.4 Snippet definition schema

```typescript
/** The type information is provided by the extension for IntelliSense in snippet files. */

export interface SnippetParameter {
    /** Parameter name — used as key in ctx.params. */
    name: string;
    /** Fixed options shown in the QuickPick. If omitted, the parameter is free-text. */
    options?: string[];
    /** Default value used when no selection is made. */
    default?: string;
}

export interface SnippetDefinition {
    /**
     * The text the user types to trigger this snippet.
     * Must start with `{` and end with `}` (e.g. `"{date}"`).
     * Contact snippets use `@` prefix instead (e.g. `"@johnsmith"`).
     */
    trigger: string;

    /** Short human-readable label shown in the autocomplete list. */
    label: string;

    /** One-line description shown as CompletionItem detail. */
    description?: string;

    /**
     * Glob pattern controlling which files this snippet appears in.
     * Matched against the workspace-relative path of the active file.
     * Example: "**\/*.md", "00-Tasks/**", "*.todo.md"
     */
    glob: string;

    /**
     * For static snippets: the literal text to insert. Mutually exclusive with `expand`.
     */
    body?: string;

    /**
     * Parameter definitions for dynamic snippets.
     * Each parameter is presented to the user via a QuickPick cascade.
     * For single-parameter snippets, the user can also provide the value by
     * selecting text before typing `{` — the selection becomes the parameter value.
     */
    parameters?: SnippetParameter[];

    /**
     * For dynamic snippets: function that produces the expansion text.
     * Receives a `SnippetContext` with document, position, params, and contacts.
     * If this function throws, the error message is inserted inline and a warning
     * notification is shown.
     */
    expand?: (ctx: SnippetContext) => string;

    /**
     * When true, this snippet can be used in path/filename expansion contexts
     * and remains active even when the snippets feature is disabled.
     * Defaults to false.
     */
    pathSafe?: boolean;
}

export interface SnippetContext {
    /** The active text document. */
    document: vscode.TextDocument;

    /** The cursor position at the time the snippet was triggered. */
    position: vscode.Position;

    /** Resolved parameter values. Keys match `SnippetParameter.name`. */
    params: Record<string, string>;

    /**
     * All resolved contacts from ContactsFeature.
     * Available regardless of whether this is a contact snippet.
     * Empty array if the contacts feature is disabled.
     */
    contacts: ResolvedContact[];
}
```

### 3.5 Trigger and autocomplete UX

#### Standard `{` snippets

1. The user types `{` in a Markdown file.
2. The `CompletionItemProvider` activates with trigger character `{`.
3. The autocomplete dropdown shows all snippets whose glob matches the current file, filtered by the text typed after `{`.
4. The user selects a snippet (e.g. `{date}`).
5. **If the snippet has no parameters**: the expanded text replaces the trigger text immediately.
6. **If the snippet has parameters**: the autocomplete dropdown closes and a QuickPick cascade begins — one QuickPick per parameter, in declaration order. After all parameters are resolved, the expanded text is inserted.
7. **Single-parameter shortcut via selection**: if the user selects text first, then types `{` and picks a single-parameter snippet, the selected text is used as the parameter value — no QuickPick appears. The trigger text plus the selection are replaced with the expansion.

#### Contact `@` snippets

1. The user types `@` in a Markdown file.
2. The `CompletionItemProvider` activates with trigger character `@`.
3. The autocomplete dropdown shows all contacts (by `id` / nickname), filtered by the text typed after `@`. Each item shows the contact's `fullName` as detail.
4. The user selects a contact (e.g. `@johnsmith`).
5. A QuickPick appears with parameter options derived from the contact record:
   - `nickname` → `"John"`
   - `full` → `"John Smith"`
   - `title` → `"John Smith (SDE 2)"`
   - `level` → `"John Smith (L2)"` — for `ColleagueContact` (no `levelId`), falls back to `full`.
   - `level full` → `"John Smith (L2) - Time in level: 9 months (from: 2026-11-20)"` — for `ColleagueContact`, falls back to `full`.
6. The selected format is inserted.

Contact snippets are **not** defined in user `.ts` files — they are generated dynamically by the extension from the `ContactsFeature` snapshot. They refresh whenever the contacts snapshot updates.

#### Contact hover

After a contact snippet is inserted (e.g. `"John Smith (L2)"`), a `HoverProvider` registered for Markdown files detects the inserted text by matching against known contact expansion values. On hover, it shows a Markdown hover card with all contact properties:

```
**John Smith**
- ID: johnsmith
- Title: Software Engineer
- Career Path: SDE
- Level: L2
- Level Start: 2026-11-20
- Pronouns: he/him
```

The hover provider maintains a map of `expandedText → contactId` for all recently inserted contact expansions within the session. This map is rebuilt whenever contacts change.

### 3.6 Todo Editor popup integration

The Todo Editor's add/edit popup (`<input>` / `<textarea>`) gains a custom autocomplete dropdown rendered within the webview:

1. When the user types `{` or `@` in the popup input, the webview sends a `{ type: 'snippetQuery', prefix: string }` message to the extension host.
2. The extension host resolves matching snippets (filtered by the collector file's glob) and returns `{ type: 'snippetSuggestions', items: SnippetSuggestion[] }`.
3. The webview renders a dropdown below the cursor position in the input, styled consistently with VS Code's autocomplete (using VS Code CSS variables).
4. Arrow keys navigate, Enter selects. Escape dismisses.
5. For parameterized snippets, the extension host handles the QuickPick cascade — the webview sends `{ type: 'snippetAccept', trigger, selectedText? }` and the extension resolves parameters via `vscode.window.showQuickPick()`, then returns `{ type: 'snippetResult', text: string }`.
6. The webview inserts the result at the cursor position in the input.

```typescript
// New message types added to ToExtensionMessage:
| { type: 'snippetQuery'; prefix: string }
| { type: 'snippetAccept'; trigger: string; selectedText?: string }

// New message types added to ToWebviewMessage:
| { type: 'snippetSuggestions'; items: SnippetSuggestion[] }
| { type: 'snippetResult'; text: string }

interface SnippetSuggestion {
    trigger: string;
    label: string;
    description?: string;
}
```

### 3.7 Path-safe snippets

Some snippets (e.g. `{date}`) are useful in programmatic contexts like filename generation or path templating during blueprint initialization. These snippets declare `pathSafe: true`.

Path-safe snippets:
- Are loaded and available even when the `snippets` feature is disabled in `.memoria/features.json`.
- Are exposed via an internal API (`SnippetsFeature.expandPathSnippet(trigger, params)`) for use by other extension components (e.g. blueprint scaffolding).
- Do **not** receive a `document` or `position` in their `SnippetContext` when invoked programmatically — these fields are `null` in that context.
- Must not depend on file context (the `document`/`position` fields). This is a convention enforced by documentation, not runtime.

### 3.8 Blueprint-shipped snippet management

- At workspace initialization, the blueprint engine copies snippet `.ts` files from the blueprint's resources into the snippet folder (e.g. `_Snippets/`). File hashes are recorded in `fileManifest`.
- Users can modify blueprint-shipped snippet files freely.
- **Reset to default**: In the VS Code Explorer, right-clicking a blueprint-shipped snippet file shows a context menu entry: **"Memoria: Reset snippet to default"**. This command:
  1. Reads the original file from the extension's bundled resources.
  2. Replaces the workspace file with the original content.
  3. Updates the `fileManifest` hash.
- The context menu entry is visible only for files that exist in `fileManifest` (i.e., blueprint-shipped files). User-created snippet files do not show this entry.
- The command is registered with a `when` clause: `resourcePath =~ /.*_Snippets.*\\.ts$/` (the folder name is dynamic, so the pattern is derived from the manifest at activation).

### 3.9 Snippet loading and lifecycle

1. **Feature start** (`snippets` enabled): The extension reads all `.ts` files in the snippet folder, compiles each via `esbuild.transform()` (TypeScript → JavaScript, ESM format), and evaluates the result via `new Function()` or dynamic `import()` of a data URI. The loaded `SnippetDefinition[]` arrays are merged into a single registry.
2. **File watching**: A `FileSystemWatcher` monitors the snippet folder for `.ts` file changes (create/modify/delete). On change, the affected file is reloaded and the registry is updated. A 300ms debounce prevents rapid re-compilation during saves.
3. **Contact snapshot updates**: When `ContactsFeature` emits an `onUpdate`, the contact snippets are regenerated from the new snapshot.
4. **Feature stop** (`snippets` disabled): The `CompletionItemProvider` and `HoverProvider` registrations are disposed. The file watcher is disposed. The snippet registry is cleared. Path-safe snippets remain loaded in a separate lightweight registry.
5. **Error handling during load**: If a `.ts` file fails to compile or evaluate, the error is logged to the OutputChannel (`Memoria`) and a warning notification is shown: `"Memoria: Failed to load snippet file '{filename}': {error}"`. Other snippet files continue loading normally.

### 3.10 Snippet expansion error handling

When a snippet's `expand()` function throws:
1. The thrown error's `.message` is captured.
2. The text `⚠️ Snippet error ({trigger}): {message}` is inserted at the cursor position.
3. A VS Code warning notification is shown: `"Memoria: Snippet '{trigger}' failed: {message}"`.

This ensures the user sees the problem both inline (where they expected text) and in the notification area.

### 4.4 Registration in extension.ts

Follows the existing `FeatureManager` callback pattern:

```typescript
const snippetsFeature = new SnippetsFeature(manifest, contactsFeature);
let snippetCompletionDisposable: vscode.Disposable | undefined;
let snippetHoverDisposable: vscode.Disposable | undefined;

featureManager.register("snippets", async (root, enabled) => {
    await snippetsFeature.refresh(root, enabled);
    if (enabled && !snippetCompletionDisposable) {
        const completionProvider = new SnippetCompletionProvider(snippetsFeature);
        snippetCompletionDisposable = vscode.languages.registerCompletionItemProvider(
            { language: "markdown", scheme: "file" },
            completionProvider,
            "{", "@",
        );
        const hoverProvider = new SnippetHoverProvider(snippetsFeature);
        snippetHoverDisposable = vscode.languages.registerHoverProvider(
            { language: "markdown", scheme: "file" },
            hoverProvider,
        );
        return;
    }
    if (!enabled && snippetCompletionDisposable) {
        snippetCompletionDisposable.dispose();
        snippetCompletionDisposable = undefined;
        snippetHoverDisposable?.dispose();
        snippetHoverDisposable = undefined;
    }
});
```

### 4.5 Completion provider

```typescript
// src/features/snippets/snippetCompletionProvider.ts

export class SnippetCompletionProvider implements vscode.CompletionItemProvider {
    constructor(private snippetsFeature: SnippetsFeature) {}

    provideCompletionItems(
        document: vscode.TextDocument,
        position: vscode.Position,
        token: vscode.CancellationToken,
        context: vscode.CompletionContext,
    ): vscode.CompletionItem[] | undefined;

    resolveCompletionItem(
        item: vscode.CompletionItem,
        token: vscode.CancellationToken,
    ): vscode.CompletionItem;
}
```

**Behavior**:
1. Reads the trigger character from `context.triggerCharacter` (`{` or `@`).
2. Retrieves all snippets from `snippetsFeature.getAllSnippets()`.
3. Filters snippets by:
   - Trigger prefix (`{` snippets vs `@` snippets).
   - Glob pattern — `vscode.languages.match()` or `minimatch` against the document's workspace-relative path.
4. Maps each matching snippet to a `CompletionItem`:
   - `kind`: `CompletionItemKind.Value`
   - `label`: snippet `label`
   - `detail`: snippet `description`
   - `filterText`: the full trigger text (e.g. `{date}`, `@johnsmith`)
   - `insertText`: empty string (actual insertion is handled by the `command` below, to support parameter resolution)
   - `command`: a `memoria.expandSnippet` command that handles QuickPick cascade and text insertion.
5. For static snippets (no `expand`, no `parameters`): `insertText` is set directly to `body` and no `command` is attached — immediate insertion.

### 4.6 Hover provider

```typescript
// src/features/snippets/snippetHoverProvider.ts

export class SnippetHoverProvider implements vscode.HoverProvider {
    constructor(private snippetsFeature: SnippetsFeature) {}

    provideHover(
        document: vscode.TextDocument,
        position: vscode.Position,
        token: vscode.CancellationToken,
    ): vscode.Hover | undefined;
}
```

**Behavior**:
1. Gets the word range at position (`document.getWordRangeAtPosition` with a broad regex).
2. Expands the range to capture multi-word contact expansions (e.g. `"John Smith (L2)"`).
3. Looks up the text in the contact expansion map maintained by `SnippetsFeature`.
4. If found, builds a `MarkdownString` with all contact properties and returns a `Hover`.
5. If not found, returns `undefined` (no hover).

### 4.7 Snippet compiler

```typescript
// src/features/snippets/snippetCompiler.ts

/**
 * Compiles a single .ts snippet file to JavaScript and evaluates it,
 * returning the exported SnippetDefinition array.
 */
export async function compileSnippetFile(
    fileUri: vscode.Uri,
    fs: vscode.FileSystem,
): Promise<SnippetDefinition[]>;
```

**Implementation**:
1. Read the `.ts` file content via `vscode.workspace.fs`.
2. Call `esbuild.transform(source, { loader: 'ts', format: 'cjs' })` to compile TypeScript to CommonJS JavaScript. Using `esbuild` is consistent with the extension's existing build pipeline.
3. Wrap the result in a module-like context and evaluate:
   ```typescript
   const module = { exports: {} as any };
   const fn = new Function("module", "exports", "require", compiledJs);
   fn(module, module.exports, createSafeRequire());
   ```
4. Extract `module.exports.default` (or `module.exports` if no default).
5. Validate the shape against `SnippetDefinition` — reject entries missing `trigger` or `glob`.
6. Return the validated array.

`createSafeRequire()` provides a limited `require` that only resolves:
- `"memoria-snippets"` → the type-only module (returns empty object at runtime; types are compile-time only).
- No access to `fs`, `child_process`, `net`, or other Node.js built-ins — snippet functions run in the extension host process and should not perform I/O.

### 4.8 Contact snippet generation

When the contacts feature is active, `SnippetsFeature` subscribes to `contactsFeature.onUpdate(snapshot)` and regenerates contact snippets:

```typescript
function generateContactSnippets(snapshot: ContactsSnapshot): SnippetDefinition[] {
    return snapshot.contacts.map(contact => ({
        trigger: `@${contact.id}`,
        label: contact.fullName,
        description: `${contact.shortTitle} — @${contact.id}`,
        glob: "**/*.md",
        parameters: [
            { name: "format", options: getContactFormatOptions(contact) },
        ],
        expand(ctx: SnippetContext): string {
            return formatContact(contact, ctx.params.format);
        },
    }));
}

function getContactFormatOptions(contact: ResolvedContact): string[] {
    const options = ["nickname", "full", "title"];
    if (contact.kind === "report") {
        options.push("level", "level full");
    }
    return options;
}

function formatContact(contact: ResolvedContact, format: string): string {
    switch (format) {
        case "nickname": return contact.nickname;
        case "full": return contact.fullName;
        case "title": return `${contact.fullName} (${contact.shortTitle})`;
        case "level":
            if (contact.kind === "report") {
                return `${contact.fullName} (${contact.resolvedCareerLevel?.key.toUpperCase() ?? "?"})`;
            }
            return contact.fullName;
        case "level full":
            if (contact.kind === "report") {
                const level = contact.resolvedCareerLevel?.key.toUpperCase() ?? "?";
                const startDate = (contact as ReportContact).levelStartDate;
                const months = monthsSince(startDate);
                return `${contact.fullName} (${level}) - Time in level: ${months} months (from: ${startDate})`;
            }
            return contact.fullName;
        default: return contact.fullName;
    }
}
```

For `ColleagueContact` (no `levelId` / `levelStartDate`), the `level` and `level full` options are omitted from the QuickPick. If a user manually provides `"level"` as a parameter, the function falls back to `full`.

### 4.9 Todo Editor integration

The Todo Editor webview gains autocomplete support for snippets in the add/edit popup:

**Webview side** (additions to `webview/main.ts`):
- Monitors `input` / `textarea` for `{` and `@` keystrokes.
- On trigger character, sends `{ type: 'snippetQuery', prefix }` to the extension.
- Renders a floating dropdown (absolutely positioned below the input cursor) with `SnippetSuggestion[]` items.
- Arrow keys navigate, Enter selects, Escape dismisses.
- On select, sends `{ type: 'snippetAccept', trigger }` to the extension.
- On `{ type: 'snippetResult', text }`, inserts the text at the current cursor position in the input and closes the dropdown.

**Extension side** (additions to `todoEditorProvider.ts`):
- Handles `snippetQuery`: calls `snippetsFeature.getAllSnippets()`, filters by prefix, returns `snippetSuggestions`.
- Handles `snippetAccept`: resolves the snippet, runs parameter QuickPicks if needed, calls `expand()`, returns `snippetResult`.

## 5. Worked examples

### 5.1 Static snippet

User types `{pp}` in `docs/notes.md`:

1. User types `{` → autocomplete shows `{pp} Private Preview`, `{PP} Public Preview`, `{date} Date`, etc.
2. User continues typing `pp` → list filters to `{pp}`.
3. User presses Enter → `{pp}` is replaced with `private preview`.

### 5.2 Dynamic snippet with QuickPick

User types `{time}` in `docs/notes.md`:

1. User types `{` → autocomplete shows all snippets.
2. User selects `{time}` → autocomplete closes.
3. QuickPick appears: `"HH" (24h)`, `"HHs" (24h + seconds)`, `"hh" (12h)`.
4. User picks `"HH"` → `{time}` is replaced with `13:10`.

### 5.3 Single-parameter snippet via selection

User selects `YYYY-MM-dd` in the document, then types `{`:

1. Autocomplete shows all snippets.
2. User selects `{date}` — which has a single parameter `format`.
3. The selected text `YYYY-MM-dd` is used as the `format` parameter value — no QuickPick appears.
4. The trigger and the selection are replaced with `2026-04-21`.

### 5.4 Contact snippet

User types `@john` in `docs/1-1-notes.md`:

1. Autocomplete shows: `@johnsmith — John Smith (SDE 2)`.
2. User selects `@johnsmith` → autocomplete closes.
3. QuickPick appears: `nickname`, `full`, `title`, `level`, `level full`.
4. User picks `level full` → `@johnsmith` is replaced with `John Smith (L2) - Time in level: 9 months (from: 2026-11-20)`.
5. User hovers over the inserted text → hover card shows all contact properties.

### 5.5 Snippet error

User defines a snippet with a bug in `_Snippets/broken.ts`:

```typescript
export default [{
    trigger: "{broken}",
    label: "Broken",
    glob: "**/*.md",
    expand() { throw new Error("division by zero"); },
}];
```

User types `{broken}` → the following text is inserted:
```
⚠️ Snippet error ({broken}): division by zero
```
A warning notification also appears: `"Memoria: Snippet '{broken}' failed: division by zero"`.

### 5.6 Todo Editor popup snippet

User clicks `"+ Add task"` in the Todo Editor and types `@john`:

1. A dropdown appears below the cursor in the popup input, showing `@johnsmith — John Smith`.
2. User selects `@johnsmith` → dropdown closes.
3. VS Code QuickPick appears (outside the webview) with format options.
4. User picks `nickname` → `@johnsmith` in the input is replaced with `John`.
5. User finishes typing the task and presses Enter to add it.

## 6. Risks & Mitigations

| Risk | Impact | Mitigation |
|---|---|---|
| User `.ts` files with malicious code execute in the extension host | High | `createSafeRequire()` blocks dangerous Node.js modules (`fs`, `child_process`, `net`, `os`, `path`, `http`, `https`). Document that snippet files run with the same trust level as any workspace code. |
| Slow snippet compilation blocks autocomplete | Medium | Compile on file save (not on every keystroke). Cache compiled results. `esbuild.transform()` is fast (~5ms per file). |
| Contact snapshot changes cause UI flicker | Low | Debounce contact snippet regeneration (500ms). Only regenerate if the contact list actually changed (shallow comparison). |
| Hover provider false positives | Low | Only match against the known expansion map. Use word-boundary-aware matching. Clear the map on workspace change. |
| User edits a blueprint-shipped snippet then resets it | Low | Reset overwrites the file completely (Git tracks history). The context menu label makes the action clear. |
| Snippet folder path collision with user content | Low | Blueprint declares the path; the conflict resolver (existing) handles re-init collisions. |
